# 뉴스/비뉴스 데이터셋 수집

`/browse` 스킬로 한국 포털을 탐색하며 방문 URL을 뉴스/비뉴스로 분류해 모은다. 이 라벨이 F1 detect 의
*gold(정답)* 가 되므로 정확도가 곧 하네스 신뢰도다.

## 왜 이렇게 하는가

detect API 는 *우리가 튜닝할 대상*이다. 그 API 로 라벨을 만들면 자기참조 오염이 생겨 95% 가 의미 없어진다.
그래서 페이지를 직접 열어 *사람처럼* 구조·내용을 보고 분류한다.

## 절차

1. **시드 무작위화:** `https://www.daum.net`, `https://www.naver.com`, `https://www.zum.com` 세 시드의
   방문 순서를 매 사이클 섞는다.
2. **탐색:** `/browse` 로 시드를 연다 → 페이지의 기사 링크/카테고리 링크를 수집 → 무작위로 따라간다(BFS/랜덤워크).
   포털 메인·섹션 인덱스도 *방문 URL*로서 분류 대상이다(대개 non-news).
3. **분류 (페이지를 직접 보고 판단):**
   - **뉴스(news):** 단일 사건·인물·정책·기업에 대한 기사 본문. 단신·속보·해설·심층 포함. 본문 narrative 가
     하나의 주제로 최소 2~3문단 이어지면 뉴스.
   - **비뉴스(non-news):** 서로 무관한 헤드라인 나열, 카테고리/이슈 인덱스, 메뉴·네비·검색 결과, 로그인,
     모금/캠페인, 자율규약·옴부즈맨 등 정책 문서, 동영상/사진 갤러리 인덱스, RSS, 회사 안내, 게시판 메인.
   - **판별 팁:** "이 페이지 본문만 떼어 읽으면 *한 기사*인가, 아니면 *여러 항목 목록*인가?"
4. **정규화·dedupe:** URL 에서 `#fragment` 와 추적 쿼리(utm_*, ref 등)를 제거한 정규형 기준 중복 제거.
   같은 기사 다른 URL(예: n.news.naver vs m.news.naver)도 1개만.
5. **다양성 유지:** 분야(정치/경제/사회/세계/IT/연예/스포츠/생활)와 매체를 고루. 한 매체·한 분야에 쏠리면 다른 섹션으로.
6. **목표:** news 100 / non-news 100. 도달까지 탐색 반복.

## 라벨 더블체크 (F1 gold 무결성)

라벨이 곧 F1 detect 의 정답이므로 *두 번 독립적으로* 판정해 일치할 때만 확정한다.
1. **1차 판정(구조):** 페이지가 단일 기사 본문 레이아웃(제목+본문 narrative)인가 vs 목록/메뉴/그리드인가.
2. **2차 판정(내용, 독립):** 본문을 읽었을 때 *하나의 사건/주제*가 2~3문단 이어지는가 vs 서로 무관한 항목 나열인가.
   *1차 결과를 보지 않고* 판단한다(확증편향 방지).
3. **합의만 확정:** 두 판정이 같으면 파일에 기록. 다르면 *재검(3차 정독)* 후에도 모호하면 **non-news 로 보수 확정**하고
   `_workspace/01_data-collector_report.md` 의 "더블체크 불일치"에 URL + 양측 사유를 남긴다.
4. **agreement rate 기록:** 1·2차 일치율을 리포트에 남긴다. <90% 면 분류 기준 자체를 점검.

> 더 강하게: 오케스트레이터가 별도 verifier 서브 에이전트에 2차 판정을 맡겨 교차 검증할 수 있다. 기본은 수집가의 2-pass.

## 출력

- `news.md`(repo 루트): 뉴스 URL 한 줄에 하나, `http`로 시작하는 절대 URL만. 주석/빈 줄 없음.
- `non-news.md`(repo 루트): 비뉴스 URL 동일 형식.
- `_workspace/01_data-collector_report.md`: 분야·매체 분포, 모호 케이스 메모, 실패(차단) URL, 총 수집 수.

> eval 스크립트가 `.split("\n")` 후 `startsWith("http")` 로 읽으므로 형식을 정확히 지킨다.

## 분류 애매 시

- 짧은 단신 vs 단순 알림이 헷갈리면: 출처·맥락·인용이 있으면 뉴스, 한두 문장 공지면 비뉴스.
- 끝내 애매하면 **non-news 로 보수 분류**하고 리포트 "모호 케이스"에 URL+이유를 남긴다(나중에 goldset-author 재검수).

## 효율

- 페이지 전체 스크린샷·전체 텍스트를 매번 길게 읽지 말 것. 제목·상단 구조·본문 첫 문단 정도로 빠르게 판단.
- 가능하면 여러 후보 링크를 모아두고 배치로 방문해 분류한다.
