# 정답(gold) 작성 — 격렬한 논의

eval-judge 채점의 *기준*을 만든다. gold 가 흔들리면 95% 측정 전체가 무의미하므로, 후보 답안을 만든 뒤
**codex 로 반대 심문**을 받아 합의에 도달한다(사용자 명시: "다른 에이전트를 소환하여 격렬히 논의").

기능별 gold 기준·단위·범위는 **반드시** `.claude/skills/newtrospect-tune/references/features.md` 를 읽고 따른다.

## 절대 원칙: 기존 프롬프트 비참조

`apps/worker/src/prompts.ts` 의 기존 기준을 보지 않는다(사용자 지시). gold 는 features.md 의 *사용자 정의*와
*뉴스 본문 자체*에서만 도출한다. 기존 프롬프트로 맞추면 그 프롬프트를 재현할 뿐 품질 향상이 아니다.

## 논의 절차 (기능별 반복)

1. **본문 정독:** 대상 뉴스 fixture 의 추출 text 를 읽는다.
2. **opus 1차 후보:** features.md 의 기준으로 gold 후보를 만든다(마킹 기능은 substring, 요약/사전지식은 자연어).
3. **codex 반대 심문:** `codex` plugin(codex:codex-rescue 서브에이전트 또는 /codex)에 후보+본문을 주고
   *반대 입장*을 요청한다. 질문 예:
   - F3 편파: "이 문장이 정말 기자의 주관/편파인가, 인용·사실 전달 아닌가?"
   - F4 중요: "이 문장들만 읽어 정말 전체 맥락이 복원되나? 빠진 인과는?"
   - F5 용어: "이 단어가 정말 중학교 교육과정 이수자에게 어려운가? 본문이 이미 풀어주지 않았나?"
   - F6 수치: "이 수치가 정말 *검색·검증 필요*한가, 단순 보도 시각/날짜 아닌가?"
   - F2 사전지식: "이 3사실이 본문 paraphrase 아닌 *기사 밖 배경*인가? 객관적 사실인가?"
   - F7 요약: "이 요약이 핵심 사건·주체·결과를 놓치지 않았나? 과장/누락은?"
4. **합의·수정:** 반박을 흡수해 gold 를 고친다. 다툼이 남으면 *본문 근거*가 강한 쪽을 택하고 `why` 에 기록.
5. **전수 작성 (샘플 금지):** gold 는 평가 대상 뉴스 *전부*(F2~F7 = 100개)에 만든다. 일부만 만들고 미루는 것은
   전수 평가 강제 규칙 위반. 배치로 효율화하되 한 건도 빠뜨리지 않는다.

## 출력 스키마

`_workspace/gold/<feature>/<fixtureId>.json`:
- F3~F6(마킹): `{ "items": [ { "text": "원문 substring", "why": "논의 근거" } ], "debatedWith": ["codex","opus"] }`
- F2(briefing): `{ "facts": [ { "fact": "...", "why": "..." } , × 3 ], "debatedWith": [...] }`
- F7(oneline): `{ "summary": "한~두 줄", "why": "...", "debatedWith": [...] }`

논의 요약 로그: `_workspace/gold/<feature>/_debate.md` (codex 가 제기한 반론과 채택/기각 결정).

## 품질 가드

- **substring 정확성:** 마킹 gold 의 `text` 는 본문에 *글자 그대로* 존재해야 한다(eval 이 매칭). 공백·따옴표까지 일치.
- **억지 금지:** 본문에 해당 유형이 거의 없으면 gold 0개가 정답이다. 칸을 채우려 일반 표현을 분류하지 않는다.
- **인용 구분(F3):** 출처·재판부·당사자 발언을 옮긴 문장은 *기자 주관이 아니므로* 제외. 기자 본인 프레이밍만.
- **codex 실패 시:** opus 단독으로 찬성/반대/심판 3역할 토론을 수행하고 `debatedWith` 에 `["opus"]` 로 표기.

## 왜 gold 와 프롬프트를 분리하나

gold 는 프롬프트와 *독립*이어야 채점이 공정하다. prompt-tuner 가 "이 케이스는 gold 가 틀렸다"고 하면,
프롬프트 편의가 아니라 *본문 근거*로만 재판단한다. gold 를 프롬프트에 맞추면 95% 는 자기충족 예언이 된다.
